# ReuseOS デザイン刷新 + 機能追加 設計書

## 目的

ランサーズの案件（リユース業向けクラウド型業務システム）獲得のため、モックアプリのデザイン品質を向上させ、募集要項に記載された機能のUIを追加する。

## 背景

- クライアントは非エンジニア。見た目の信頼感が受注に直結する
- 日本のBtoB SaaSはライトテーマが主流。ダークモードは不安感を与える
- 募集要項の必須確認ポイント（古物営業法対応・本人確認・LINE/SMS連携）のUIが未実装

---

## 1. カラーテーマ刷新

### 方針
- ダークモード完全廃止
- 白ベース + 薄いグレーで清潔感のあるデザイン
- 黒（#000）は使わない

### カラーパレット

| 用途 | 色 | 値 |
|------|------|------|
| 背景（メイン） | White | #FFFFFF |
| 背景（サブ） | Slate 50 | #F8FAFC |
| サイドバー背景 | White | #FFFFFF + border-right |
| テキスト（主） | Slate 800 | #1E293B |
| テキスト（副） | Slate 500 | #64748B |
| アクセント | Indigo 600 | #4F46E5 |
| アクセント（ホバー）| Indigo 700 | #4338CA |
| アクセント（薄） | Indigo 50 | #EEF2FF |
| 成功 | Emerald 500 | #10B981 |
| 警告 | Amber 500 | #F59E0B |
| エラー | Rose 500 | #F43F6E |
| ボーダー | Slate 200 | #E2E8F0 |

### 変更対象ファイル
- `tailwind.config.ts` — カスタムカラー定義
- `src/app/globals.css` — CSS変数の更新、dark mode関連の削除
- `src/app/layout.tsx` — dark classの削除
- `src/app/(dashboard)/layout.tsx` — サイドバーの背景色変更

## 2. デザイン品質向上

- カードのシャドウ: `shadow-sm` → `shadow-sm` + `border` で軽やかに
- バッジの色味をソフトトーンに統一
- ボタンのホバーエフェクト改善
- 余白・フォントサイズの微調整

---

## 3. 電子買受書プレビュー

### 画面
- `/purchases/[id]` 詳細画面に「買受書を表示」ボタン追加

### ダイアログ内容（PDF風レイアウト）
- タイトル: 「買受書」
- 古物商許可番号
- 取引日
- 相手方情報（氏名・住所・生年月日）
- 品目テーブル（品名・数量・金額）
- 合計金額
- 確認方法（運転免許証等）
- 担当者署名欄

---

## 4. 古物台帳ビュー

### ルート
- `/ledger` — サイドバーに「古物台帳」として追加

### テーブルカラム（法定記載事項）
- 受入番号
- 取引年月日
- 区分（買受/委託）
- 品目
- 数量
- 特徴
- 相手方（住所・氏名・職業・年齢）
- 確認方法
- 確認書類番号

### モックデータ
- 既存のpurchase + customerデータから自動生成風に表示

---

## 5. 本人確認書類UI

### 画面
- `/customers/[id]` 詳細画面に「本人確認書類」セクション追加

### UI要素
- 書類タイプ（運転免許証・マイナンバーカード等）の表示
- ダミーの書類画像カード（表面・裏面）
- 確認ステータスバッジ（確認済/未確認/期限切れ）
- 確認日・有効期限の表示
- 「書類をアップロード」のドロップエリア風UI

---

## 6. LINE/SMS連携UI

### 画面
- `/customers/[id]` 詳細画面に「メッセージ」セクション追加

### UI要素
- チャネル切替タブ（LINE / SMS）
- 送信履歴テーブル（日時・内容・ステータス）
- 「メッセージを送信」ボタン → ダイアログ（テンプレート選択 + テキスト入力）
- LINE連携ステータス表示（連携済/未連携）

### モックデータ
- 3-5件の送信履歴（買取完了通知、査定結果連絡等）
